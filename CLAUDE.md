# Claude 行为准则与项目级指令 (Project Instructions)

这是 `finance-tracker` 项目专属的 Claude 行为规范。当你在这个目录下工作时，必须严格遵守以下约定。

## 🤖 全自动多智能体前端验证与修复 (Autonomous Multi-Agent UI Verification)

**【核心原则】：绝不让人类用户充当“浏览器预览器”、“环境 Debug 助手”或“配置修理工”。**
当用户要求你修改、添加或重构任何前端代码（特别是 UI 组件、页面布局、CSS/Tailwind 类名）后，你必须执行全自动验证，**且在遇到任何问题（无论是代码报错、样式不符，还是运行环境挂了）时，都必须自行或召唤子 Agent 解决，绝对禁止向用户求助**。

### 强制执行流程

1. **执行自动化截图与测试**：
   只要你修改了前端代码，必须主动执行以下脚本：
   ```bash
   cd /home/lyy/AI/app/finance-tracker
   python scripts/verify_ui.py http://localhost:3000/你的目标路由
   ```
   *注：脚本内已配置自动清理僵尸端口、删除锁死缓存以及长超时等待按需编译。*

2. **多模态自我审查 (Multi-modal Review)**：
   脚本运行完毕后，使用 `Read` 工具读取生成的 `ui_logs.txt` 和 `ui_screenshot.png`。
   - 检查日志是否有 Error（React Hydration, 404, 语法错误，或者 Playwright 超时等运行环境错误）。
   - 像挑剔的设计师一样检查截图：白屏？错位？重叠？未渲染的数据？

3. **【关键】多智能体委派与环境自愈 (Self-Healing & Multi-Agent Delegation)**：
   **CRITICAL: 如果你在步骤 1 或 2 中发现了任何错误（不管是代码缺陷，还是测试脚本抛出的 ERR_CONNECTION_RESET、端口冲突等环境错误），绝对禁止在对话中向人类用户解释错误并询问“是否需要修复”。**

   - **如果是环境/配置错误**：你必须立即使用 Bash 工具自己查杀进程（如 `kill -9`）、清理缓存或修改配置，直到环境恢复。
   - **如果是前端代码错误或视觉瑕疵**：你必须立即使用 **`Agent` 工具**（子智能体类型选择 `general-purpose`），将修复任务完全委派给另一个 Claude 分身。

   在调用 Agent 时，你必须在 Prompt 中提供：
   - 当前的上下文、报错内容（甚至是环境报错）或你发现的视觉瑕疵。
   - 明确指示子 Agent：“**你是专门的 UI 修复和环境调试专家。请你读取代码或排查环境并修复问题。修复后，你必须亲自再次运行 `python scripts/verify_ui.py` 并读取新截图，确认无误。如果还是没修好，请你自己在后台继续改、继续测，不要停下。直到日志干净、截图完美后，再向我返回最终的成功报告。**”

4. **最终汇报**：
   只有当你自己确认一切正常，或者子 Agent 经历了多轮后台修改测试并返回给你一个“完美修复”的结果后，你才能向人类用户回复。回复内容应包含：“任务完成，我（或与审查 Agent 协作）已经完成了前端多轮测试与修复，当前页面渲染完美且无报错。”
