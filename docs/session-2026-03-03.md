### 七、功能层面三项改进

#### 7.1 周期性记录（定期账单）
- **Schema**：新增 `RecurringTransaction` 模型（含 `dayOfMonth`、`isActive`、`lastApplied` 字段）
- `src/lib/actions/recurring.ts`：
  - `getRecurringTransactions` — 查询列表
  - `createRecurringTransaction` — 创建（含校验）
  - `deleteRecurringTransaction` — 删除（userId 防越权）
  - `toggleRecurringTransaction` — 启用/暂停切换
  - `applyDueRecurringTransactions` — 核心：访问收支页时自动检查本月到期记录，通过 `lastApplied` 月份判断避免重复触发
- `RecurringDialog.tsx` — 管理弹窗（增删启停，展示上次触发时间）
- `transactions/page.tsx` 集成：页面加载时自动执行触发检查

#### 7.2 收支记录分页
- `TransactionList.tsx` 新增：`PAGE_SIZE = 20`、`page` state
- 筛选条件变化时通过 `useEffect` 自动重置到第1页
- 底部分页控件（超过一页才显示）：上一页 / 第 X / 共 Y 页 / 下一页

#### 7.3 多月预算对比（近3个月）
- `getBudgetComparison(monthCount=3)` server action：
  - 并行查预算表 + 交易明细，手动按月分组（Prisma groupBy 不支持按月）
  - 返回 `{ months: string[], rows: [{categoryName, months: [{label, budget, spent}]}] }`
  - 自动补查只有交易、没有预算的分类名称
- `budgets/page.tsx` 底部新增对比表格：超出预算的实际金额标红，无数据显示「—」

## 关键技术决策

| 决策 | 原因 |
|------|------|
| `applyDueRecurringTransactions` 在页面加载时调用（非 cron） | 无服务器定时任务权限，访问即触发足以满足日常使用 |
| 多月对比手动按月分组而非 SQL GROUP BY | Prisma/SQLite groupBy 不支持按日期字段的月份分组 |
| `TransactionList` 客户端筛选+分页 | 数据量不大，避免每次筛选都发请求，体验更流畅 |
| `<React.Fragment key={...}>` 代替 `<>` | `.map()` 内需要 key，短语法不支持 key prop |
